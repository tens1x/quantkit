# Research

论文、策略想法、复盘笔记的存放处。这里的内容不随代码重写而丢失。

## 目录结构

```
research/
├── papers/              # 论文笔记（每篇一个 md）
├── strategies/          # 策略想法（从论文/思考中提炼）
├── notes/               # 月度复盘 + 零散笔记
│   └── YYYY-MM-review.md    # 月度复盘笔记
└── README.md
```

## 论文笔记模板

文件名: `papers/YYYY-MM-DD-简短标题.md`

```markdown
# 论文标题

- **来源**: 链接 / 期刊 / arXiv ID
- **日期**: 阅读日期
- **主题**: e.g. 因子投资 / 动量策略 / 风险管理

## 一句话总结
这篇论文在说什么

## 关键发现
- 发现 1
- 发现 2
- 发现 3

## 对 QuantKit 的启发
- [ ] 可以加什么因子？→ 描述
- [ ] 可以改什么策略？→ 描述
- [ ] 风险评估需要加什么？→ 描述

## 我的疑问
- 没想明白的地方
- 需要进一步研究的点

## 下一步
- [ ] 如果要实现，转到 strategies/ 写策略文档
- [ ] 如果只是认知更新，记录在 notes/ 里
```

## 策略想法模板

文件名: `strategies/策略名.md`

```markdown
# 策略名称

- **来源**: 哪篇论文 / 哪个想法 / 哪次复盘
- **类型**: 因子 / 择时 / 风控 / 组合优化
- **优先级**: 高 / 中 / 低

## 逻辑（大白话）
用一段话描述这个策略在做什么，为什么可能有效

## 买入条件
- 条件 1
- 条件 2

## 卖出条件
- 条件 1
- 条件 2

## 需要的数据
- 已有: e.g. OHLCV, PE, ROE
- 缺的: e.g. 历史 PE 时间序列, 行业分类

## 预期与风险
- 什么市场环境下有效？
- 什么情况下会失效？
- 最大风险是什么？

## 实现 Checklist
- [ ] 策略逻辑确认（你自己想清楚了）
- [ ] 数据源确认（需要的数据都能拿到）
- [ ] 告诉 Claude Code 实现
- [ ] Codex 编码 + 测试
- [ ] 回测结果记录
- [ ] 决定：留下 / 弃掉 / 改进

## 回测结果（实现后填写）
- 测试标的:
- 时间范围:
- Total Return:
- vs Buy & Hold:
- Max Drawdown:
- Sharpe:
- 结论:
```

## 工作流

详见 `PLAN.md` 的两条主线。简要版：

```
读论文 → papers/笔记
    ↓
提炼策略 → strategies/文档
    ↓
喂给 Claude Code → Codex 实现 → 回测验证
    ↓
月度复盘 → notes/复盘笔记
    ↓
发现新问题 → 读下一篇论文...
```
