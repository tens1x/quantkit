# Research

论文、策略想法、学习笔记的存放处。喂进来的东西最终会转化为代码。

## 目录结构

```
research/
├── papers/          # 论文笔记（每篇一个 md）
├── strategies/      # 策略想法（从论文或自己的思考中提炼）
├── notes/           # 零散笔记、灵感、疑问
└── README.md
```

## 论文笔记模板

在 `papers/` 下创建 md 文件，建议格式：

```markdown
# 论文标题

- **来源**: 链接 / 期刊 / arXiv ID
- **日期**: 阅读日期
- **主题**: e.g. 因子投资 / 动量策略 / 风险管理

## 核心观点
一两句话概括这篇论文在说什么

## 关键发现
- 发现 1
- 发现 2

## 对 QuantKit 的启发
- 可以加什么因子？
- 可以改什么策略？
- 风险评估需要加什么维度？

## 疑问
- 没想明白的地方
```

## 策略想法模板

在 `strategies/` 下创建 md 文件：

```markdown
# 策略名称

- **来源**: 哪篇论文 / 哪个想法启发的
- **类型**: 因子 / 择时 / 风控 / 组合优化

## 逻辑
用大白话描述这个策略在做什么

## 买入条件
- 条件 1
- 条件 2

## 卖出条件
- 条件 1

## 需要的数据
- 已有的: e.g. OHLCV, PE, ROE
- 缺的: e.g. 历史 PE 时间序列, 行业分类, 财报日期

## 预期
- 这个策略在什么市场环境下有效？
- 什么情况下会失效？

## 实现状态
- [ ] 数据准备
- [ ] 策略编码
- [ ] 回测验证
- [ ] 加入 CLI 菜单
```

## 工作流

```
读论文 → 写笔记 (papers/)
    ↓
提炼策略想法 (strategies/)
    ↓
告诉 Claude Code "实现这个策略"
    ↓
Codex 编码 → 回测验证 → 合入 QuantKit
    ↓
用真实持仓跑一遍 → 记录发现 (notes/)
    ↓
读下一篇论文...
```

程序随时可以重写，认知的积累在这个目录里。
